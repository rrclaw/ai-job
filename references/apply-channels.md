# 投递通道手册（自动化分级）

调研+实战日期 2026-07。核心原则：**能全自动的全自动，有封号/合规风险的只做半自动，最终提交动作永远留给候选人确认**（学 AIHawk 教训：取其定制，弃其海投）。

## 通道三档

### A 档：邮箱直投（可全自动到「草稿就绪」）
- 适用：公司公开招聘邮箱（如 DeepSeek talent@ 系）
- 流程：按公司要求的主题格式（如「姓名-岗位-联系方式」）自动生成邮件 + 三句话正文（一句定位、两条硬证据、作品集链接）+ 附简历 PDF → 建草稿 → 候选人过目后点发送
- 自动化工具：Gmail MCP `create_draft`（支持 base64 附件 ≤25MB）/ lark-cli mail / 任意 SMTP
- 为什么停在草稿：①外发动作需候选人最终确认 ②发件邮箱要与简历联系邮箱一致或可解释，机器不该替人选发件身份
- 命名规范：附件「姓名-岗位-简历.pdf」，正文不超过 150 字

### B 档：官方招聘门户（半自动：agent 填表 + 人过验证码和提交）
- 适用：Moka（app.mokahr.com）、飞书招聘（*.jobs.feishu.cn）、北森、公司官网表单
- 流程：browser automation（如 claude-in-chrome）打开职位页 → 候选人完成手机验证码登录 → agent 自动填表（基本信息/经历/上传 PDF）→ 候选人目检 → 提交
- 卡点：手机验证码必须真人；部分门户有防自动化检测，操作节奏放慢
- 一次登录后同门户多岗位可连投

### C 档：直聊平台（只做辅助，不做自动）
- 适用：Boss直聘、猎聘、脉脉
- 红线：自动打招呼/批量投递工具违反平台条款，封号风险真实存在，且开场白模板痕迹会被 HR 的 AI 助手识别
- 正确做法：skill 只负责生成「mini 简历开场白」（一句定位+两条硬证据+一个链接，见 tracker 模板），候选人手动粘贴发送
- 开场白本身会被 HR 侧 AI 读取，按 LLM 初筛标准写

## 投递纪律

1. **精准投递，禁海投**：每个岗位用对应定制版简历（交叉投会掉 10-20 分，见审阅实测）
2. 每家投递前 30 分钟内跑一次 checks.py --links，确保作品集链接全部在线（面试官点开 404 = 自爆）
3. 投递动作完成后立刻回填 tracker.md（日期/渠道/版本/主题行），凭证截图存 output/
4. 同公司多岗位间隔 ≥48h，避免被判海投
5. 邮箱直投的发件地址尽量与简历联系邮箱一致；不一致时正文末尾注明「简历联系方式为准」

## 各档产物

- `candidates/<name>/apply_plan.md`：目标公司清单（渠道/优先级/简历版本/主题行/状态）
- 邮件草稿（A 档）/ 预填表单会话（B 档）/ 开场白文本（C 档）
